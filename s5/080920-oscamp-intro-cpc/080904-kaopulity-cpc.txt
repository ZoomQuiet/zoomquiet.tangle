靠谱的蟒营
~ 追忆首届金山训练营Python 特训班

= 缘起. =
靠谱: Kaopulity ,是从FreeBSD 社区流传出来的,好程序员的综合素质体现,是任何公司都渴望使用的人才品质;
蟒营: Kingsoft College Python Camp,是金山训练营Python 特训班的简称;
  隨着金山的涉入互联网业务,要求进行敏捷的服务开发,所以,金山一些部门开始使用Python 来加速开发和维护;而且当代中国没有任何一个高等院校有能力进行Python 开发知识的教育;金山训练营作为金山人才预备仓库,认识到了这一情况,主动发起在训练营学期内部开办有针对性开发技能的特训班的倡议;

= 承.接 =
  笔者作为CPyUG~Chinese Python User Group,中国Python 用户组的发起人,有多年Python 推广/开发/培训的经验,训练营领导一提,就非常高兴的承接下来;不过,没有照搬学校的那种上讲下听的形式,而是结合了Python 技术社区的体验,针对学生仔的特质,和公司用人的基本素质要求,设计了一套"教学",尝试在短短4周时间的实战项目开发中发掘出具有靠谱潜力的学员来!
  从今年7月中旬动念,到8月1日,训练营暑期开班,纠集同为CPyUG社区行者的几名同学,高速完成了:
0. 蟒营的"教学"形式和流程: 全真式的项目开发活动,完全交给学员自发形成的团队,蟒营只进行技术答疑和成果验收的组织;
1. 蟒营教案: 导入介绍,开发基础,项目协同,协同环境等等;
2. 蟒营环境: 借用主机,重装系统,开辟虚拟机,预装系统;开辟专用列表,组织蟒营维基,等等;
3. 蟒营课题: 收集了一批实用工具的开发需求,将各个部门需要,但是又无法挤出人力来开发的系统,都拿出来准备让学员自行认领开发;

就等学员报名入我觳来也 ;-)

= 转,战! =
  8月1日,是蟒营宣讲和报名日. 必须在最短时间内让通过训练营初试的学员们:
A: 金山有哪些部门需要Python 人才
B: 大约了解什么是Python
C: 大约了解知道什么是蟒营
  然后,才可能吸引足够多的学员来开坛讲经,,,
  幸好,使用了经过考验的幻灯资料(在北京讲过多次的资料),先后有20名学员报名掺合蟒营.好的,一切按照计划来:
i:  2小时Python 扫盲;
ii: 半小时蟒营本质解析~挑选可用人才,贡献实用软件;
iii:一小时团队协同导入,说明版本/项目/知识管理的概念,支持系统,管理技巧;
iv: 组织每天早晨下午各一次一刻钟的站立会议,要求每个人口头通报当日计划完成/已经完成/问题困难;
v:  组织每周2/5中午的答疑,要求提前一日汇总问题反馈给导师们
vi: 组织每周5上午的一次作品演示,要求学员将开发的作品部署到指定域名的主机环境中演示出来
  学生仔们面对完全陌生的知识/流程/环境/行为/要求等等时的反应,就是源自本身最真实的能力/素质/性格的反应,也因此可以观察对比出潜力的大小来;而蟒营的持续评分,就是基于学员对各种指令的反应形式和速度,进行评比的;

= 好!散? =
  "导师团可以提供任何帮助,但是无法主动提供~因为我们不知道你们需要什么帮助!"
  这是笔者在头一个星期反复再三不断殷切的提示学生亿们说的话;
  但是!居然没有人相信...嗬嗬嗬,这真的是出乎我们的意外;
  所以,学员产品了"导师在折磨学生"的猜想,而且有退出蟒营活动的行为;
  当然,更多的学员被激起了牛劲:"好!说这是简单的事儿,那我就一定自主解决了,就不问!反正只会说: TiosnG!"
TiosnG! ~ There is one sited named Google!
  这也也社区创造出来的专用词,是"提问的智慧"一文的精简建议;因为学生仔在漫长的学校学习过程中由于太容易获得答案,或是答案的重要性很低,所以,根本就没足够的诚意对问题本身进行探寻:
0. 是否主动搜索过,问题是否有相关的开源软件可以解决?
1. 是否主动和项目主持人或是社区接触过?咨询基本情况?
2. 是否主动下载过代码尝试运行?
3. 是否主动和同学讨论过?
4. 是否主动分析过问题由哪些方面组成,是否都是自个儿不知道的?
5. 是否主动整理自个儿的知识,明白自个儿缺少什么?知道自个儿喜欢/愿意什么方面的开发?
6. 是否主动就问题的解决设计过一些方案?对方案是否有对比和测试?
...
  以上尝试 < 60% 的,基本就是没有尽自己能力去解决问题,而习惯性的想转嫁问题给其它人的,是不靠谱(unkaopulity)的!所以,蟒营,最本质的述求就是期望在最短的时间内将学生气质最大限度的清除掉,灌入职员的思路:
 * 凡事计划为先,让问题及早暴露;
 * 做事证据为要,让一切工作有案可寻;
 * 困难多方尝试,学习必须自!帮助得用诚意和创意来换取,职场中没有免费的午餐!

  Piu~~的一下,首届蟒营,短暂的四周就结束了,5个作品组,有3个达到基本可用的地步,让学员自个儿都惊叹自个儿的潜力无穷;这期间,看到学员开始使用邮件列表进行大范围的讨论,注意有学员主动认领团队的额外职责,观察到学员自主加班追进度,也发觉学员开展相互主动分享模块...
  居然,真的没有学员再咨询Python 开发的技术问题,而多是操作系统环境的配置,项目的管理方法,对外的沟通技巧等等公司职员真正应该理解和掌握的技巧;可见 Python 的确是真真好学易用的哪...
  可惜粥少僧多,训练营每期毕业学员是全珠海所有事业部共同来瓜分的;笔者所在 KSPEG ~ KingSoft Process Enhacnement Group ,金山过程改进组 招聘的过程改进工程师,除了对技术的要求,对管理和沟通都有一定要求,最终在首期蟒营中仅仅确认了3名学员,也算是善终;而且积累了大量录音/邮件/常见问题,为下次开营作好了充分储备!
  到8月29日,蠎营最后一次作品演示后,在当晚举行了K歌欢送会;一直在讲靠谱,目标是挑选靠谱人才的,应该靠谱的蟒营,至此,算是正式闭营;不过,相关工程环境,沟通渠道,都没有关闭,蟒营作品在继续发展中,蟒营理念也在完善中,蟒营学员也在持续沟通自学中...
  笔者及其导师团伙,期待下届蟒营,可以更加合理,高效的进行灌输,改造出靠谱的金山人来!
